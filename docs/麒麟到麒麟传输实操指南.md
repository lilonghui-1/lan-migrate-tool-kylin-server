## 一、概述

本指南介绍如何在 **麒麟系统（Kylin OS）** 之间进行文件传输，支持：
- **桌面版 ↔ 桌面版**：使用 GUI 图形界面
- **桌面版 ↔ 服务器版**：GUI客户端 + 命令行服务端
- **服务器版 ↔ 服务器版**：纯命令行操作

### 系统架构
```
┌─────────────────┐         TCP 9000          ┌─────────────────┐
│  麒麟桌面版 GUI  │  ═══════════════════════► │  麒麟桌面版 GUI  │
│   (PyQt6)       │   双向传输协议             │   (PyQt6)       │
│                 │ ◄═══════════════════════  │                 │

┌─────────────────┐         TCP 9000          ┌─────────────────┐
│ 麒麟桌面版 GUI  │  ═══════════════════════► │ 麒麟服务器版 CLI │
│   (PyQt6)       │   双向传输协议             │   (命令行)       │
│                 │ ◄═══════════════════════  │                 │
└─────────────────┘                            └─────────────────┘

┌─────────────────┐         TCP 9000          ┌─────────────────┐
```

## 三、麒麟桌面版 ↔ 麒麟桌面版

### 3.1 环境准备

**两台麒麟桌面版都需要安装 PyQt6：**

```bash
# 检查 Python 版本
python3 --version  # 需要 3.8+

# 安装 PyQt6
pip3 install PyQt6

# 安装 zeroconf（可选，用于设备发现）
pip3 install zeroconf
```

### 3.2 启动服务端（麒麟B）

在麒麟B上启动服务端：

```bash
cd ~/lan-migrate-tool

# 启动服务端
python3 server/kylin_server.py -p 9000

# 或后台运行
python3 server/kylin_server.py --daemon
```

输出：
```
[INFO] 麒麟服务端启动成功，端口: 9000
[INFO] 接收目录: ~/LAN_Migrate_Received
[INFO] 等待连接...
```

### 3.3 启动客户端（麒麟A）

在麒麟A上启动 GUI 客户端：

```bash
cd ~/lan-migrate-tool

# 启动 GUI 客户端
python3 kylin_gui_client.py
```

### 3.4 连接与传输

1. **发现设备**：
   - 客户端会自动扫描局域网内的设备
   - 麒麟B会显示为 `"麒麟服务端"`

2. **连接设备**：
   - 点击麒麟B的设备卡片
   - 或手动输入麒麟B的IP地址

3. **传输操作**：
   - 连接成功后自动切换到 **"麒麟传输"** 页面
   - 点击 **"浏览远程目录"** 查看麒麟B的文件系统
   - 点击 **"上传文件到远程"** 发送文件到麒麟B
   - 在远程目录浏览器中 **选中文件** → **"下载选中文件"** 从麒麟B下载

### 3.5 双向传输示例

**麒麟A → 麒麟B（上传）：**
```
麒麟A: 点击"上传文件到远程"
       → 选择本地文件 /home/user/document.pdf
       → 在远程目录浏览器中选择保存目录 /home/user/Documents
       → 点击"选择此目录"
       → 文件上传到麒麟B
```

**麒麟B → 麒麟A（下载）：**
```
麒麟A: 点击"浏览远程目录"
       → 导航到 /home/user/Downloads
       → 选中 file.tar.gz
       → 点击"下载选中文件"
       → 选择本地保存目录 ~/Downloads
       → 文件下载到麒麟A
```

## 四、麒麟桌面版 ↔ 麒麟服务器版

### 4.1 麒麟桌面版 → 麒麟服务器版

#### 步骤1：服务器端启动服务端

在麒麟服务器版上：

```bash
cd ~/lan-migrate-tool

# 启动服务端
python3 server/kylin_server.py -p 9000

# 或使用 systemd 服务
sudo systemctl start kylin-server
```

#### 步骤2：桌面版启动客户端

在麒麟桌面版上：

```bash
cd ~/lan-migrate-tool

# 启动 GUI 客户端
python3 kylin_gui_client.py
```

#### 步骤3：连接与传输

- 在 GUI 中手动输入服务器IP地址
- 使用"麒麟传输"页面进行双向传输

### 4.2 麒麟服务器版 → 麒麟桌面版

#### 步骤1：桌面版启动服务端

在麒麟桌面版上：

```bash
cd ~/lan-migrate-tool

# 启动服务端
python3 server/kylin_server.py -p 9000
```

#### 步骤2：服务器版使用命令行客户端

在麒麟服务器版上：

```bash
cd ~/lan-migrate-tool

# 连接到桌面版
python3 server/kylin_cli_client.py 192.168.1.100 -p 9000

# 进入交互模式
```

#### 步骤3：命令行传输操作

```bash
# 列出远程目录
kylin> ls /home/user/Documents

# 上传文件到桌面版
kylin> upload /root/data.tar.gz /home/user/Downloads

# 下载文件到服务器版
kylin> download /home/user/report.pdf ~/Downloads

# 上传整个目录
kylin> uploaddir /var/log /home/user/logs
```

## 五、麒麟服务器版 ↔ 麒麟服务器版

### 5.1 环境准备

两台服务器都不需要 GUI，只需 Python 3.8+：

```bash
# 检查 Python
python3 --version

# 无需安装 PyQt6
```

### 5.2 启动服务端（服务器B）

```bash
cd ~/lan-migrate-tool

# 启动服务端
python3 server/kylin_server.py -p 9000

# 或使用 systemd
sudo systemctl start kylin-server
```

### 5.3 使用命令行客户端（服务器A）

#### 方式一：交互模式

```bash
python3 server/kylin_cli_client.py 192.168.1.200 -p 9000 -i
```

#### 方式二：单命令模式

```bash
# 列出远程目录
python3 server/kylin_cli_client.py 192.168.1.200 --ls /home/user

# 上传单个文件
python3 server/kylin_cli_client.py 192.168.1.200 --upload /root/file.txt --remote-dir /home/user/backup

# 上传整个目录
python3 server/kylin_cli_client.py 192.168.1.200 --uploaddir /var/log --remote-dir /home/user/logs

# 下载文件
python3 server/kylin_cli_client.py 192.168.1.200 --download /home/user/data.tar.gz --local-dir ~/Downloads
```

### 5.4 批量传输脚本

创建批量传输脚本：

```bash
#!/bin/bash
# batch_upload.sh

SERVER_IP="192.168.1.200"
PORT=9000

# 上传多个文件
for file in /data/*.tar.gz; do
    python3 server/kylin_cli_client.py $SERVER_IP -p $PORT --upload "$file" --remote-dir /backup
done

echo "批量上传完成"
```

## 六、命令行客户端详细使用

### 6.1 命令行参数

```
用法: python3 server/kylin_cli_client.py <server_ip> [选项]

必需参数:
  server_ip         远程服务端 IP 地址

可选参数:
  -p, --port PORT   服务端端口 (默认: 9000)
  --upload FILE     上传单个文件
  --uploaddir DIR   上传整个目录
  --download FILE   下载单个文件
  --remote-dir DIR  远程保存目录 (用于上传)
  --local-dir DIR   本地保存目录 (用于下载，默认 ~/Downloads)
  --ls [PATH]       列出远程目录 (不指定路径则列出家目录)
  -i, --interactive 进入交互模式
  -h, --help        显示帮助信息
```

### 6.2 交互模式命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `ls [path]` | 列出远程目录 | `ls /home/user` |
| `upload <file> [remote_dir]` | 上传文件 | `upload /root/file.txt /backup` |
| `uploaddir <dir> [remote_dir]` | 上传目录 | `uploaddir /var/log /logs` |
| `download <file> [local_dir]` | 下载文件 | `download /data/file.tar.gz ~/Downloads` |
| `help` | 显示帮助 | `help` |
| `quit` | 退出 | `quit` |

### 6.3 使用示例

#### 示例1：备份服务器数据到另一台服务器

```bash
# 服务器A → 服务器B
python3 server/kylin_cli_client.py 192.168.1.200 --uploaddir /var/log --remote-dir /backup/logs

python3 server/kylin_cli_client.py 192.168.1.200 --uploaddir /etc --remote-dir /backup/config
```

#### 示例2：从远程服务器下载日志文件

```bash
# 交互模式
python3 server/kylin_cli_client.py 192.168.1.200 -i

kylin> ls /var/log
kylin> download /var/log/syslog ~/logs
kylin> download /var/log/auth.log ~/logs
```

#### 示例3：定时自动备份

创建 cron 任务：

```bash
# 编辑 crontab
crontab -e

# 添加定时任务（每天凌晨2点备份）
0 2 * * * /home/user/lan-migrate-tool/scripts/auto_backup.sh
```

脚本内容：
```bash
#!/bin/bash
# auto_backup.sh

cd /home/user/lan-migrate-tool
python3 server/kylin_cli_client.py 192.168.1.200 --uploaddir /data --remote-dir /backup/$(date +%Y%m%d)
```

## 七、常见问题排查

### 7.1 PyQt6 安装失败（桌面版）

**问题**：pip3 install PyQt6 报错

**解决方案**：
```bash
# 安装系统依赖
sudo apt update
sudo apt install python3-pyqt6

# 或使用 pip
pip3 install PyQt6 --user
```

### 7.2 连接超时

**问题**：命令行客户端连接超时

**排查步骤**：
```bash
# 1. 检查网络
ping 192.168.1.200

# 2. 检查端口
telnet 192.168.1.200 9000

# 3. 检查防火墙
sudo ufw allow 9000/tcp
sudo firewall-cmd --add-port=9000/tcp --permanent
sudo firewall-cmd --reload

# 4. 检查服务端是否运行
ps aux | grep kylin_server
sudo netstat -tlnp | grep 9000
```

### 7.3 权限问题

**问题**：无法写入文件

**解决方案**：
```bash
# 修改接收目录权限
chmod 755 ~/LAN_Migrate_Received

# 或修改所有者
sudo chown -R $USER:$USER ~/LAN_Migrate_Received
```

### 7.4 大文件传输中断

**问题**：传输大文件时中断

**解决方案**：
- 检查网络稳定性
- 使用断点续传（重新连接后继续）
- 增加超时时间（修改 config.py 中的 SOCKET_TIMEOUT）

### 7.5 中文文件名乱码

**问题**：中文文件名显示乱码

**解决方案**：
```bash
# 检查系统编码
echo $LANG

# 应为 zh_CN.UTF-8 或类似值

# 如果不是，设置编码
export LANG=zh_CN.UTF-8
```

### B. systemd 服务配置

```ini
[Unit]
Description=LAN Migrate Kylin Server
After=network.target

[Service]
Type=simple
User=kylin
WorkingDirectory=/home/kylin/lan-migrate-tool
ExecStart=/usr/bin/python3 /home/kylin/lan-migrate-tool/server/kylin_server.py -p 9000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```
